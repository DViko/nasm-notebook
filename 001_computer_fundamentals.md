# Архитектура компьютера

> **Архитектура компьютера** — это организация, структура и поведение компьютерной системы, включая взаимодействие аппаратного и программного обеспечения.

### CPU:

> **Процессор** выполняет инструкции программы, управляет обработкой данных и координирует работу компонентов системы.

- **Ключевые компоненты процессора:**
  - **Control Unit (CU):** управляет работой процессора.
  - **Arithmetic Logic Unit (ALU):** выполняет арифметические и логические операции.
  - **Registers:** небольшие и очень быстрые ячейки памяти процессора для немедленного доступа к данным. Используются для хранения операндов, адресов памяти, указателей стека и результатов вычислений.

### Memory:

> **Память** обычно подразделяется на два типа:

- **Primary Memory (RAM):**
  - Временно хранит используемые данные и программы.
  - Быстрый доступ (медленнее регистров, но быстрее устройств хранения).
  - Энергозависимая **volatile memory:** данные теряются при отключении питания.

- **Secondary Memory (Storage):**
  - Сохраняет данные на длительное время.
  - Медленный доступ.
  - Энергонезависимая **non-volatile memory:** данные сохраняются при отключении питания.

### Bus:

> **Шина** — это система связи, которая передает данные между компонентами внутри или вне компьютера.

- **Основные типы шин:**
  - **Address Bus:**  передаёт адрес памяти или устройства, к которому выполняется обращение.
  - **Control Bus:** передает сигналы управления (чтение или запись).
  - **Data Bus:** передает фактические данные, которые должны быть записаны или считаны.

- **Как работает шина (на примере чтения из памяти):**
  - **Address Bus:**
    - Процессор размещает на адресной шине адрес памяти, к которой необходимо обратиться.

  - **Control Bus:**
    - Процессор размещает сигнал на шине управления, указывая хочет ли он выполнить операцию чтения или записи.
      - **Read:** Процессор активирует сигнал чтения, после чего память помещает данные на шину данных.
      - **Write:** Процессор активирует сигнал записи, после чего данные с шины данных записываются в память.

  - **Data Bus:**
    - Содержит фактические данные которые должны быть записаны в память или считаны из памяти.
      - **Read operation:** Значение по адресу целевой памяти помещается на шину данных которая затем считывается процессором.
      - **Write operation:** Значение которое необходимо записать по целевому адресу памяти помещается на шину данных.

## Memory Hierarchy

> **Иерархия памяти** это различные уровни памяти в компьютере каждый из которых имеет разную скорость и емкость для хранения данных.

- **Register:**
  - **Функция:**
    - Хранение операндов, адресов, указателей стека.
    - Обеспечивает максимально быстрый доступ к данным, необходимым для арифметических и логических операций, управления потоком выполнения и обработки данных.
  - **Расположение:** Внутри процессора.
  - **Скорость:** Самый быстрый тип памяти.
  - **Размер:** Очень маленький (десятки байт).

- **Cache Memory:**
  - **Функция:**
    - Хранит часто используемые данные и команды, сокращая время доступа к данным в оперативной памяти.
    - **Организация:**
      - **L1 Cache:** Самый маленький и быстрый.
      - **L2 Cache:** Больше по размеру и немного медленнее чем L1.
      - **L3 Cache:** Ещё больше и медленнее, общий для ядер процессора.
  - **Расположение:** Обычно расположен на кристалле процессора.
  - **Скорость:** Очень быстрая.
  - **Размер:** Небольшой, от нескольких КБ до нескольких МБ.

- **Main Memory (RAM):**
  - **Функция:**
    - Хранение текущих данных и инструкций.
  - **Расположение:** Вне процессора (на материнской плате).
  - **Скорость:** Быстрая.
  - **Размер:** Несколько ГБ.

- **Secondary Storage:**
  - **Функция:**
    - Используется для бессрочного хранения данных.
  - **Расположение:** Вне процессора.
  - **Скорость:** Медленная.
  - **Размер:** От нескольких сотен ГБ до нескольких ТБ.

- **Tertiary Storage:**
  - **Функция:**
    - Используется для резервного копирования и архивирования данных.
  - **Расположение:** Внешние устроства хранения.
  - **Скорость:** Самый медленный тип памяти.
  - **Размер:** Может быть очень большим.

## Endianness

> **Эндианность** это порядок хранения или передачи байтов в памяти. Это имеет значение для понимания того, как представляются данные в ассемблере и как процессор обращается к многобайтовым типам данных.

**Два основных типа порядка байтов:**

- **Big-Endian:**
  - Most Significant Byte (MSB) (старший байт) хранится по младшему адресу в памяти.
  - Least Significant Byte (LSB) (младший байт) хранится по старшему адресу в памяти.
  - Пример: 32-битное целое число ` 0x12345678 `, в памяти:
  ```text
    | Адрес | Значение |
    | :---- | :------- |
    | 0x00  | 0x12     |
    | 0x01  | 0x34     |
    | 0x02  | 0x56     |
    | 0x03  | 0x78     |
  ```

- **Little-Endian:**
  - Least Significant Byte (LSB) (младший байт) хранится по младшему адресу в памяти.
  - Most Significant Byte (MSB) (старший байт) хранится по старшему адресу в памяти.
  - Пример: 32-битное целое число ` 0x12345678 `, в памяти:
  ```text
    | memory address | value stored at the address |
    | :------------- | :-------------------------- |
    | 0x00           | 0x78                        |
    | 0x01           | 0x56                        |
    | 0x02           | 0x34                        |
    | 0x03           | 0x12                        |
  ```

### Интерпретация данных:

- **Интерпретация данных:**
  - Порядок байтов влияет на то как данные интерпретируются при чтении из памяти.
  - Программа считывающая данные в одном порядке байтов, может неправильно интерпретировать данные если они были записаны в другом порядке байтов.

- **Сетевые протоколы:**
  - Многие сетевые протоколы используют формат **big-endian** (также известный как ` Network Byte Order `) для обеспечения согласованности между различными системами.
  - При передаче данных по сетям системам необходимо преобразовывать свой исходный порядок байтов в порядок байтов **big-endian** для корректной работы.

- **Кроссплатформенная совместимость:**
  - Порядок байтов может вызывать проблемы при кроссплатформенной разработке программного обеспечения.
  - Если одна система использует порядок байтов **little-endian**, а другая **big-endian**, то структуры данных должны быть соответствующим образом преобразованы при совместном использовании между системами.

> **Заметка:** Архитектура x86/x86-64 использует ` Little-Endian `.

## Instruction Set Architecture (ISA)

> **ISA** —  это интерфейс между программным и аппаратным обеспечением, определяющий набор инструкций, которые может выполнять процессор.

### Ключевые компоненты архитектуры:

- **Набор инструкций:**
  - Совокупность инструкций, которые может выполнять процессор, включая арифметические, логические операции, управление потоком выполнения, перемещение данных и многое другое.
  - Инструкции часто подразделяются на следующие типы:
    - **Data Movement Instructions:** Перемещают данные между регистрами, памятью и устройствами ввода-вывода (` MOV, PUSH, POP, LOAD, STORE `).
    - **Arithmetic Instructions:** Выполняют математические операции (` ADD, SUB, MUL, DIV `).
    - **Logical Instructions:** Выполняют побитовые операции (` AND, OR, NOT, XOR `).
    - **Control Flow Instructions:** Изменяют ход выполнения инструкций (` JMP, CALL, RET `).
    - **Input/Output Instructions:** Управляют передачей данных на периферийные устройства и с них (` IN, OUT `).

- **Режимы адресации:**
  - Методы используемые для указания операндов для инструкций.
  - Основные типы адресации:
    - **Register:** Операнд в регистре: ` mov eax, ebx `.
    - **Indirect:** Адрес находится в регистре: ` mov eax, [ebx] `.
    - **Immediate** Операнд — константа в самой инструкции: ` mov eax, 5 `.
    - **Direct(Memory):** Адрес операнда указан напрямую: ` mov eax, [0x1234] `.
    - **Indexed:** Базовый адрес, индекс и смещение: ` mov eax, [ebx + ecx*4] `.

- **Регистры:**
  - Небольшие, быстродействующие ячейки памяти внутри центрального процессора, используемые для временного хранения данных во время выполнения инструкций.
  - Основные типы регистров:
    - **General-Purpose:** Используются для различных операций с данными (` RAX, RBX, RCX, RDX ` ...).
    - **Special-Purpose:** Используются для выполнения определенных функций (` RSP, RBP ` ...).

- **Форматы инструкций:**
  - Структура инструкции обычно включает следующие поля:
    - **OpCode:** Указывает операцию которая должна быть выполнена.
    - **Operands:** Указывают данные подлежащие обработке, которые могут включать регистры, непосредственные значения или адреса памяти.
    - **Addressing Mode:** Указывает как интерпретировать операнды.

- **Управляющие сигналы:**
  - Сигналы блока центрального процессора для управления выполнением инструкций, включая чтение или запись в память, активацию арифметико-логического устройства и управление потоком выполнения.


### Типы архитектур наборов инструкций:

- **CISC** (Complex Instruction Set Architecture)
  - Содержит большое количество инструкций, включая сложные инструкции, которые могут выполнять несколько операций за одну инструкцию.
  - ` x86 architecture (Intel, AMD) `

- **RISC** (Reduced Instruction Set Architecture)
  - Содержит меньшее количество инструкций. Инструкции обычно проще и более однотипны по времени выполнения.
  - Особое внимание уделяется архитектуре загрузки/сохранения, где доступ к памяти ограничен конкретными инструкциями ` LOAD/STORE `.
  - ` ARM, MIPS architecture `

- **VLIW** (Very Long Instruction Word)
  - Использует длинные инструкции, объединяющие несколько операций в один блок.
  - Компилятор отвечает за планирование инструкций, которые должны выполняться одновременно, параллельно.
  - ` Itanium architecture `

- **EPIC** (Explicitly Parallel Instruction Computing)
  - Расширение ` VLIW `. Разработан для использования параллелизма на уровне инструкций путем явного указания параллельной операции в наборе инструкций.
  - Компилятор отвечает за планирование инструкций, которые должны выполняться одновременно, параллельно.
  - ` Intel's Itanium `


### Важность ISA

- **Совместимость:** Определяет совместимость программного обеспечения. Программы написанные для одной архитектуры набора команд, могут не работать на другой без перекомпиляции или эмуляции.
- **Производительность:** Влияет на характеристики производительности центрального процессора и системы в целом.
- **Программирование:** Определяет принципы проектирования компиляторов и разработки языков программирования, поскольку устанавливает как инструкции высокого уровня преобразуются в машинный код.

## Interrupts and Exceptions

> **Прерывания и исключения** это механизмы используемые процессором для обработки событий нарушающих нормальный ход выполнения программы. И то и другое позволяет процессору реагировать на такие события как сигналы оборудования или ошибки. Однако они различаются по причинам возникновения и способам обработки.

### Прерывания:

> Аппаратные или программные сигналы, указывающие на то что событие требует немедленного внимания.
Процессор временно приостанавливает текущую задачу для обработки прерывания, а затем возобновляет её выполнение.

**Типы прерываний:**
  - **Hardware Interrupts:**
    - Внешние устройства (клавиатура, мышь, сетевая карта).
    - Нажатие клавиши на клавиатуре отправляет прерывание процессору, позволяя ему обработать событие.
    - Наиболее распространенные области применения:
      - Операции ввода-вывода
      - Таймеры
      - Системные события

  - **Software Interrupts:**
    - Запускается при выполнении специальной инструкции в программе, часто для системных вызовов или специальных функций.
    - Примером может служить системный вызов для запроса услуги у операционной системы, например чтения или записи файлов.

### Процесс обработки прерываний:

При возникновении прерывания процессор выполняет следующие действия:

- **Сохранить состояние процесса:**
  - Регистры, счетчик команд текущей выполняемой задачи.
- **Обработать прерывание (ISR):**
  - Передача управления обработчику прерывания.
- **Обработать событие:**
  - Пример: чтение данных с устройства ввода-вывода или обработку события таймера.
- **Восстановить состояние:**
  - Процессор восстанавливает сохраненное состояние и возобновляет прерванную задачу.

### Характеристики прерываний:

- **Asynchronous:** Прерывания могут происходить в любой момент выполнения программы.
- **External or software-triggered:** Прерывания вызываются внешними устройствами или специальными инструкциями программного обеспечения.
- **Prioritized:** Прерывания могут иметь разные уровни приоритета.

### Исключения:

> **Исключения** — это события возникающие в результате выполнения инструкций, часто из-за ошибки или особых условий которые необходимо обработать процессору или операционной системе.

**Типы исключений:**

- **Trap:**
  - Генерируется программным обеспечением преднамеренно, как правило посредством системного вызова или другой инструкции.
  - Программа может использовать перехватчик (` trap `) для запроса услуги у операционной системы (выделение памяти, обработка файлов).

- **Fault:**
  - Возникают из-за ошибок при выполнении инструкций, таких как доступ к недопустимому адресу памяти (ошибка сегментации).
  - Операционная система или обработчик исключения могут попытаться исправить проблему, если это удаётся, программа может возобновить работу. В противном случае программа завершается часто с сообщением об ошибке.

- **Abort:**
  - Возникает в результате серьезных аппаратных или системных ошибок, таких как ошибки четности памяти или неисправности оборудования.
  - Как правило это необратимо и приводит к завершению работы программы или сбою системы.

### Процесс обработки исключений:

При возникновении исключения обычно выполняются следующие действия:

- **Сохранение текущего состояния:** Процессор сохраняет состояние программы (регистры, счетчик команд).
- **Обработчик исключений:** Процессор переходит к обработчику исключений который обрабатывает конкретное исключение.
- **Обработка исключения**: Обработчик может попытаться решить проблему.
- **Возобновление или завершение**: В зависимости от типа исключения, программа может либо возобновить выполнение, либо быть завершена.

### Характеристики исключений:

- **Synchronous:** Исключения возникают непосредственно во время выполнения инструкций.
- **Instruction-related:** Исключения связаны с конкретной выполняемой инструкцией или состоянием процессора.

### Ключевые различия между прерываниями и исключениями:

```text
    | Характеристика       | Прерывание                                        | Исключение                                                                       |
    | :------------------- | :------------------------------------------------ | :------------------------------------------------------------------------------- |
    | Источник             | Внешнее устройство или программная инструкция     | Внутреннее событие процессора                                                    |
    | Момент возникновения | Асинхронно (может произойти в любой момент)       | Синхронно (во время выполнения инструкции)                                       |
    | Причина              | События ввода-вывода, аппаратные сигналы          | Ошибки инструкций, системные вызовы                                              |
    | Обработка            | Используется обработчик прерываний (ISR)          | Используется обработчик исключений                                               |
    | Приоритет            | Часто имеет уровни приоритета                     | Обычно требует немедленной обработки                                             |
    | Восстановление       | Выполнение программы обычно может быть продолжено | Некоторые исключения позволяют продолжить выполнение, другие завершают программу |
```
