# Архитектура компьютера

> **Архитектура компьютера** структура и поведение компьютерной системы. Определяет возможности, производительность и взаимодействие между аппаратными компонентами.

### CPU:

> **Процессор** мозг компьютера, осуществляет все коммуникации и выполняет инструкции.

- **Ключевые компоненты процессора:**
  - **Control Unit (CU):** Управляет работой процессора.
  - **Arithmetic Logic Unit (ALU):** Выполняет арифметические и логические операции.
  - **Registers:** Небольшие и очень быстрые ячейки памяти процессора, используемые для немедленного доступа к данным.

### Memory:

> **Память ** в системе обычно подразделяется на два типа:

- **Primary Memory (RAM):**
  - Временно хранит используемые данные и программы.
  - Быстрый доступ (медленнее чем регистры но быстрее чем устройства хранения данных)
  - Является **volatile** типом.
  - Все данные загруженные в память, будут потеряны после выключения системы.

- **Secondary Memory (Storage):**
  - Сохраняет данные на длительное время.
  - Медленный доступ.
  - Является **non-volatile** типом.
  - Все данные сохраняются при выключении системы.

### Bus

> **Шина** система связи, которая передает данные между компонентами внутри или вне компьютера.

- **Основные типы шин:**
  - **Address Bus:** Передаёт информацию о том, куда следует записывать или загружать данные.
  - **Control Bus:** Передает сигналы, связанные с операциями управления (например, сигналы чтения или записи).
  - **Data Bus:** Передает фактические данные, которые должны быть записаны или загружены.

- **Как работает шина:**
  - **Address Bus:**
    - Процессор размещает адрес памяти к которой он хочет обратиться, на адресной шине. Шина передает адрес ячейки памяти из которой должны быть считаны или записаны данные.

  - **Control Bus:**
    - Процессор посылает управляющий сигнал по шине управления, указывая, хочет ли он выполнить операцию чтения или записи.
      - **Read:** Шина управления даст указание шине памяти отправить данные обратно процессору.
      - **Write:** Шина управления даст указание шине памяти принимать данные от процессора.

  - **Data Bus:**
    - Содержит фактические данные, которые должны быть записаны в память или считаны из памяти.
      - **Read operation:** Значение по адресу целевой памяти помещается на шину данных, которая затем считывается процессором.
      - **Write operation:** Значение которое необходимо записать по целевому адресу памяти, помещается на шину данных.

## Иерархия памяти

> **Иерархия памяти** различные уровни памяти в компьютере, каждый из которых имеет разную скорость и емкость для хранения данных.

- **Register:**
  - **Функция:**
    - Сохраняет текущие данные и инструкции, которые обрабатывает процессор.
    - Обеспечивают максимально быстрый доступ к данным, необходимым для арифметических и логических операций, управления потоком выполнения и обработки данных.
  - **Расположение:** Внутри процессора.
  - **Скорость:** Самый быстрый тип памяти.
  - **Размер:** Очень маленький.
  - **Категория:** Основная.

- **Cache Memory:**
  - **Функция:**
    - Хранит часто используемые данные и команды, сокращая время доступа к данным в оперативной памяти (ОЗУ).
    - **Организация:**:
      - **L1 Cache:** Самый маленький и быстрый, встроен в ядро ​​процессора.
      - **L2 Cache:** Больше по размеру и немного медленнее чем L1, может располагаться на процессоре или рядом с ним.
      - **L3 Cache:** Ещё больше и медленнее, используется несколькими ядрами процессора.
  - **Расположение** В непосредственной близости от ЦП (обычно на том же или соседнем чипе)
  - **Скорость** Быстрее чем оперативная память, но медленнее чем регистры.
  - **Размер** Небольшой (обычно от нескольких КБ до нескольких МБ).
  - **Категория** Основная.

- **Main Memory (RAM):**
  - **Функция:**
    - Данные и инструкции, используемые в данный момент, загружаются в оперативную память.
  - **Расположение:** Вне процессора.
  - **Скорость:** Медленнее, чем кэш, но быстрее, чем вторичное хранилище.
  - **Размер:** Обычно несколько ГБ.
  - **Категория:** Основная.

- **Secondary Storage:**
  - **Функция:**
    - Используется для бессрочного хранения данных.
  - **Расположение:** Вне процессора.
  - **Скорость:** Медленнее, чем у оперативной памяти.
  - **Размер:** Обычно от нескольких сотен ГБ до нескольких ТБ.
  - **Категория:** Вторичная.

- **Tertiary Storage:**
  - **Функция:**
    - Используется для резервного копирования и архивирования данных.
  - **Расположение:** Вне основных устройств хранения.
  - **Скорость:** Самый медленный тип памяти.
  - **Размер:** Может быть очень большим.
  - **Категория:** Вторичная.

## Endianness

**Эндианность** порядок хранения или передачи байтов в памяти. Это имеет значение для понимания того, как представляются данные в ассемблере и как процессор обращается к многобайтовым типам данных.

**Два основных типа порядка байтов:**

- **Big-Endian:**
  - **Small end** (младший байт) хранится по старшему адресу.
  - **Big end** (старший байт) хранится по младшему адресу в памяти.
  - Например: есть 32-битное целое число, ` 0x12345678 ` оно будет храниться в памяти следующим образом:
  ```text
    | memory address | value stored at the address |
    | :------------- | :-------------------------- |
    | 0x00           | 12                          |
    | 0x01           | 34                          |
    | 0x02           | 45                          |
    | 0x03           | 56                          |
    | 0x04           | 78                          |
  ```

- **Little-Endian:**
  - **Small end** (младший байт) хранится по младшему адресу в памяти.
  - **Big end** (старший байт) хранится по старшему адресу в памяти.
  - Например: есть 32-битное целое число, ` 0x12345678 ` оно будет храниться в памяти следующим образом:
  ```text
    | memory address | value stored at the address |
    | :------------- | :-------------------------- |
    | 0x00           | 78                          |
    | 0x01           | 56                          |
    | 0x02           | 34                          |
    | 0x03           | 12                          |
  ```

### Интерпретация данных:

- **Интерпретация данных:**:
  - Порядок байтов влияет на то как данные интерпретируются при чтении из памяти.
  - Программа считывающая данные в одном порядке байтов, может неправильно интерпретировать данные если они были записаны в другом порядке байтов.

- **Сетевые протоколы:**:
  - Многие сетевые протоколы используют формат **big-endian** (также известный как ` Network Byte Order `) для обеспечения согласованности между различными системами.
  - При передаче данных по сетям системам необходимо преобразовывать свой исходный порядок байтов в порядок байтов **big-endian** для корректной работы.

- **Кроссплатформенная совместимость**:
  - Порядок байтов может вызывать проблемы при кроссплатформенной разработке программного обеспечения.
  - Если одна система использует порядок байтов **little-endian**, а другая **big-endian**, то структуры данных должны быть соответствующим образом преобразованы при совместном использовании между системами.

## Instruction Set Architecture (ISA)

> **Архитектура набора инструкций:** Часть компьютерной архитектуры определяющая набор инструкций, которые может выполнять процессор. Служит интерфейсом между программным и аппаратным обеспечением, позволяя разрабатывать программы способные взаимодействовать с центральным процессором.

### Ключевые компоненты архитектуры:

- **Набор инструкций:**
  - Совокупность инструкций, которые может выполнять центральный процессор, включая арифметические, логические операции, управление потоком выполнения, перемещение данных и многое другое.
  - Инструкции часто подразделяются на следующие типы:
    - **Data Movement Instructions:** Перемещают данные между регистрами, памятью и устройствами ввода-вывода (` LOAD, STORE `).
    - **Arithmetic Instructions:** Выполняют математические операции (` ADD, SUB, MUL, DIV `).
    - **Logical Instructions:** Выполняют побитовые операции (` AND, OR, NOT, XOR `).
    - **Control Flow Instructions:** Изменяют ход выполнения инструкций (` JUMP, CALL, RETURN `).
    - **Input/Output Instructions:** Управляет передачей данных на периферийные устройства и с них.

- **Режимы адресации:**
  - Методы используемые для указания операндов для инструкций.
  - К распространенным режимам адресации относятся:
    - **Immediate Addressing:** Операнд указывается непосредственно в инструкции.
    - **Direct Addressing:** Адрес операнда указывается напрямую.
    - **Indirect Addressing:** Адрес операнда задается регистром или ячейкой памяти.
    - **Indexed Addressing:** Адрес операнда генерируется путем сложения постоянного значения со значением регистра.
    - **Register Addressing:** Операнд находится в определенном регистре.

- **Регистры:**
  - Небольшие, быстродействующие ячейки памяти внутри центрального процессора, используемые для временного хранения данных во время выполнения инструкций.
  - К типам регистров относятся:
    - **General-Purpse Registers:** Используются для различных операций с данными (` R0, R1, R2 ` ...).
    - **Special-Purpse Registers:** Используются для выполнения определенных функций (` RSP, RBP ` ...) .

- **Форматы инструкций:**
  - Структура инструкции обычно включает следующие поля:
    - **OpCode:** Указывает операцию которая должна быть выполнена.
    - **Operands:** Указывают данные подлежащие обработке, которые могут включать регистры, непосредственные значения или адреса памяти.
    - **Addressing Mode:** Указывает как интерпретировать операнды.

- **Управляющие сигналы:**
  - Сигналы генерируемые блоком управления центрального процессора для управления выполнением инструкций, включая чтение или запись в память, активацию арифметико-логического устройства и управление потоком выполнения.


### Типы архитектур наборов инструкций:

- **CISC** (Complex Instruction Set Architecture)
  - Содержит большое количество инструкций, включая сложные инструкции, которые могут выполнять несколько операций за одну инструкцию.
  - ` x86 architecture (Intel, AMD) `

- **RISC** (Reduced Instruction Set Architecture)
  - Содержит меньшее количество инструкций, каждая из которых предназначена для выполнения за один цикл.
  - Особое внимание уделяется архитектуре загрузки/сохранения, где доступ к памяти ограничен конкретными инструкциями загрузки и сохранения.
  - ` ARM architecture, MIPS architecture `

- **VLIW** (Very Long Instruction Word)
  - Использует длинные инструкции, объединяющие несколько операций в один блок.
  - Компилятор отвечает за планирование инструкций, которые должны выполняться одновременно, параллельно.
  - ` Itanium architecture `

- **EPIC** (Explicitly Parallel Instruction Computing)
  - Разработан для использования параллелизма на уровне инструкций путем явного указания параллельной операции в наборе инструкций.
  - Компилятор отвечает за планирование инструкций, которые должны выполняться одновременно, параллельно.
  - ` Intel's Itanium `


### Важность ISA

- **Совместимость:** Определяет совместимость программного обеспечения. Программы написанные для одной архитектуры набора команд, могут не работать на другой без перекомпиляции или эмуляции.
- **Производительность:** Влияет на характеристики производительности центрального процессора и системы в целом.
- **Программирование:** Определяет принципы проектирования компиляторов и разработки языков программирования, поскольку устанавливает как инструкции высокого уровня преобразуются в машинный код.

## Прерывания и исключения

> **Прерывания и исключения:** Механизмы используемые процессором для обработки событий нарушающих нормальный ход выполнения программы. И то и другое позволяет процессору реагировать на такие события как сигналы оборудования или ошибки. Однако они различаются по причинам возникновения и способам обработки.

### Прерывания:

> Сигналы посылаемые аппаратным или программным обеспечением, указывающие на то, что событие требует немедленного внимания.
Центральный процессор временно приостанавливает текущую задачу для обработки прерывания, а затем возобновляет её выполнение.

**Типы прерываний:**
  - **Hardware Interrupts:**
    - Генерируется внешними устройствами (такими как клавиатура, мышь или сетевая карта) для подачи сигнала процессору о необходимости внимания.
    - Например, нажатие клавиши на клавиатуре отправляет прерывание в центральный процессор, позволяя ему обработать нажатие клавиши.
    - Наиболее распространенные области применения:
      - операции ввода-вывода
      - Таймеры
      - Системные события

  - **Software Interrupts:**
    - Запускается при выполнении специальной инструкции в программе, часто для системных вызовов или специальных функций.
    - Примером может служить системный вызов для запроса услуги у операционной системы, например чтения или записи файлов.

### Процесс обработки прерываний:

При возникновении прерывания центральный процессор выполняет следующие действия:

- Сохранить текущее состояние:
  - Центральный процессор сохраняет состояние (регистры и счетчик команд) текущей выполняемой задачи.
- Обработать прерывание (ISR):
  - Процессор переходит к специальной функции называемой ` interrupt service routine (ISR) ` или ` interrupt handler `, которая отвечает за обработку прерывания.
- Обработайть событие:
  - Обработчик прерываний обрабатывает событие, например чтение данных с устройства ввода-вывода или обработку события таймера.
- Восстановить состояние:
  - После завершения обработки прерывания процессор восстанавливает сохраненное состояние и возобновляет прерванную задачу.

### Характеристики прерываний:

- **Asynchronous:** Может происходить в любой момент выполнения программы.
- **Prioritized:** Прерывания могут иметь разные уровни приоритета, то есть прерывания с более высоким приоритетом могут прерывать прерывания с более низким приоритетом.

### Исключения:

**Исключения:** События возникающие непосредственно в результате выполнения инструкций, часто из-за ошибки или особых условий которые необходимо обработать центральному процессору или операционной системе.

**Типы исключений:**

- **Traps:**
  - Генерируется программным обеспечением преднамеренно, как правило посредствам системного вызова или другой инструкции, предназначенной для запуска операционной системы.
  - Например: программа может использовать перехватчик (` trap `) для запроса услуги у операционной системы, такой как выделение памяти или обработка файлов.

- **Faults:**
  - Возникают из-за ошибок при выполнении инструкций, таких как доступ к недопустимому адресу памяти (ошибка сегментации) или деление на ноль.
  - Процессор пытается исправить проблему. Если ему это удаётся, программа может возобновить работу. В противном случае программа завершается часто с сообщением об ошибке.

- **Aborts:**
  - Возникает в результате серьезных аппаратных или системных ошибок, таких как ошибки четности памяти или неисправности оборудования.
  - Как правило это необратимо и приводит к завершению работы программы или сбою системы.

### Процесс обработки исключений:

При возникновении исключения обычно выполняются следующие действия:

- **Сохранение текущего состояния:** Процессор сохраняет состояние программы (регистры, счетчик команд).
- **Обработчик исключений:** Процессор переходит к обработчику исключений который обрабатывает конкретное исключение.
- **Обработка исключения**: Обработчик может попытаться решить проблему.
- **Возобновление или завершение**: В зависимости от типа исключения, программа может либо возобновить выполнение, либо быть завершена.

### Характеристики исключений:

- **Synchronous:** Исключения напрямую связаны с выполнением конкретных инструкций, то есть они возникают в ответ на определенные операции.
- **Non-maskable:** В отличие от некоторых прерываний, исключения обычно не могут быть проигнорированы процессором и должны обрабатываться немедленно.

### Ключевые различия между прерываниями и исключениями:

```text
    | Feature        | Interrupt                             | Exception                                                     |
    | :------------- | :------------------------------------ | :------------------------------------------------------------ |
    | Source         | External (hardware or software)       | Internal (CPU instruction execution)                          |
    | Timing         | Asynchronous(can happen anytime)      | Synchronous (occurrs during instruction execution)            |
    | Cause          | I/O events, hardware signals          | Instruction errors, system calls                              |
    | Handling       | Uses interrupt service routines (ISR) | Uses exception handlers                                       |
    | Priority       | Often prioritized                     | Usually handled immediately                                   |
    | Recoverability | Often recoverable (resume program)    | Some exceptions are recoverable, others terminate the program |
```
